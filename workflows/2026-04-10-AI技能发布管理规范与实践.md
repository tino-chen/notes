# AI技能发布管理规范

## 这是什么

AI 技能发布到 ClawHub 时的质量规范和踩坑记录。避免在公开技能中泄露敏感信息、发布不规范的元数据。

## 最终效果

- 每个技能都有完整的溯源信息（GitHub 仓库 + ClawHub 页面）
- 展示名称统一为中文用户友好的格式
- 发布前自动检查敏感信息，不会再把私人信息推到公开仓库

## 整体架构

```
技能开发完成
    │
    ├─ 1. 检查 frontmatter 元数据（repository + clawhub 必填）
    ├─ 2. 检查展示名称（Title Case + 中文名）
    ├─ 3. 敏感信息扫描（grep 脱敏）
    └─ 4. 发布
         ├─ clawhub publish → ClawHub
         └─ git push → GitHub
```

## 实操步骤

### 第一步：检查元数据

SKILL.md frontmatter 必须包含以下字段：

```yaml
---
name: skill-name
description: 技能功能描述
repository: https://github.com/用户名/仓库名称
clawhub: https://clawhub.com/用户名/技能名称
---
```

验证：
- `repository` 指向实际存在的 GitHub 仓库
- `clawhub` 指向 ClawHub 页面

### 第二步：检查展示名称

面向中文用户时，展示名称采用「Title Case（中文名称）」格式：

```yaml
title: "Workflow Note（流程笔记）"
```

- ✅ `Workflow Note（流程笔记）`
- ❌ `workflow-note`、`WorkflowNote`

### 第三步：敏感信息扫描

发布前对 SKILL.md 和所有文档文件运行扫描：

```bash
grep -rn "真实姓名\|邮箱\|open_id\|credentials\|~/Library" ~/.openclaw/skills/skill-name/
```

命中任何一条 → 用 `xxx` 替代，保留上下文可读性：

```markdown
# 错误
"不暴露 xxx 真实身份"

# 修正
"保护用户隐私，不泄露个人身份信息"
```

### 第四步：发布

```bash
# 发布到 ClawHub
clawhub publish

# 验证 ClawHub 页面展示正确
```

## 踩坑记录

### 1. 敏感信息泄露到公开技能

**问题**：发布 `workflow-note` 技能时，SKILL.md 中包含 `"不暴露 xxx 真实身份"` 等私人信息。

**原因**：技能文件中残留了内部配置规则，发布前没有做敏感信息扫描。

**解决**：立即撤回 → 脱敏 → 重新发布版本 1.2.1。此后每次发布前强制执行 grep 扫描。

### 2. 展示名称缺少中文

**问题**：ClawHub 上技能只显示英文 `workflow-note`，中文用户无法理解用途。

**原因**：frontmatter 的 `title` 字段没有包含中文名称。

**解决**：更新为 `title: "Workflow Note（流程笔记）"`，发布版本 1.2.2。

### 3. `isolated: true` 字段无效

**问题**：在技能 frontmatter 中写了 `isolated: true`，以为能让技能在独立会话中运行。

**原因**：该字段在技能系统中被忽略，只在 cron 任务的 `sessionTarget` 属性中有效。

**解决**：搜索 OpenClaw 和 ClawHub CLI 源码确认后，从所有技能中删除该字段。教训：不要依赖文档中未明确说明的配置。
